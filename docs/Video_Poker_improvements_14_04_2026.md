# 1. ВСЕ РЕЖИМЫ — ОБЩИЕ ПРАВКИ

### 1.1 Иконка выхода — отступ слева
- **Баг:** иконка `table_exit.svg` прижата к левому краю экрана, нет отступа
- **Фикс:** добавить left margin/padding, чтобы иконка была ровно на уровне левого края контролбара (control bar внизу)
- **Файлы:** `game.gd`, `multi_hand_game.gd`, `spin_poker_game.gd` — везде, где создаётся кнопка выхода

### 1.2 Эффект нажатия кнопок (press effect)
- **Сейчас:** кнопки не дают визуального фидбэка при нажатии (кроме HOLD на картах, где есть эффект)
- **Нужно:** при нажатии любой кнопки — эффект "продавливания" (scale down). Реализация: при `pressed` → `scale = Vector2(0.93, 0.93)`, при `released` → tween обратно к `Vector2(1, 1)` за 0.1с
- **Область:** ВСЕ кнопки во ВСЕХ режимах: DEAL, DRAW, BET, MAX BET, denomination, выход, магазин "+", настройки, кнопки в popup'ах (STAY/LEAVE, YES/NO, OK, GOT IT), кнопки линий в Spin
- **Исключение:** карты (у них уже есть HOLD toggle эффект)

### 1.3 Переключение БАЛАНС ↔ ИГР (Credits mode)
- **Сейчас:** при переключении БАЛАНС на ИГР меняется только поле баланса
- **Нужно:** при переключении в режим ИГР (Credits) менять ВСЕ три поля:
  - БАЛАНС → значение в играх (credits = chips / denomination)
  - ВЫИГРЫШ → значение в играх
  - ОБЩАЯ СТАВКА → значение в играх
- Кнопку ставки внизу (BET) **НЕ менять** — игрок должен видеть сколько фишек списывается базово
- **Логика:** `credits_value = chips_value / current_denomination`. Если denomination = 1, то chips = credits
- **Область:** применить во ВСЕХ режимах (game.gd, multi_hand_game.gd, spin_poker_game.gd)
- Переключение уже существует (кнопка БАЛАНС/ИГР в top bar). Нужно расширить логику на WIN и TOTAL BET

### 1.4 Кнопки ставки — независимость друг от друга
- **Баг:** после первого раунда кнопка BET дизейблится (корректно), но если нажать "Уровень ставки" или "Макс ставка" — BET разлочивается. Кнопки не должны зависеть друг от друга
- **Фикс:** в стейте DEALING/DRAWING/EVALUATING все три кнопки (BET, BET LEVEL/denomination, MAX BET) должны быть `disabled = true` независимо. Разлочка — только при переходе в стейт IDLE (ожидание нового раунда)
- **Область:** ВСЕ режимы. Проверить `_on_state_changed()` — там не должно быть взаимных зависимостей между кнопками

### 1.5 Z-order окон и popup'ов
- **Баг:** подсказки ("Сделай ставку", "Выберите карты и жмите DRAW") и множители (Ultra VP) отображаются **поверх** popup'ов (диалог выхода, окно правил)
- **Фикс:** все popup'ы (exit confirm, rules, double confirm, shop) должны иметь `z_index` выше ВСЕХ игровых элементов. Рекомендация:
  - Игровое поле, карты: z_index = 0
  - Подсказки, бейджи, множители: z_index = 10–20
  - Popup overlay (затемнение): z_index = 50
  - Popup content: z_index = 51
- **Область:** ВСЕ режимы

### 1.6 Окно правил — закрытие по тапу вне окна
- **Сейчас:** окно правил закрывается только при нажатии на X
- **Нужно:** закрытие при нажатии в ЛЮБОМ месте экрана за пределами окна (по тапу на overlay/background)
- **Реализация:** полноэкранный прозрачный `Button` или `Control` за popup'ом, `pressed.connect(_close_rules)`
- **Область:** ВСЕ режимы (game.gd, multi_hand_game.gd, spin_poker_game.gd)

### 1.7 Дизейбл кнопки BET — приглушение текста
- **Баг:** когда кнопка BET переходит в disabled (после DEAL), текст на ней (количество фишек) остаётся ярким, как у активной кнопки
- **Фикс:** при `_bet_amount_btn.disabled = true` → также приглушать `modulate.a` текста (или менять цвет шрифта на dimmed). При разлочке — восстанавливать
- **Область:** ВСЕ режимы

# 2. SINGLE HAND

### 2.1 Шрифт Royal Flush растёт при каждом DEAL
- **Баг:** при каждом нажатии на DEAL увеличивается размер шрифта строки максимального выигрыша (Royal Flush, выделенная красным) в paytable. Высота блока paytable растёт, выталкивается вниз, перекрывает другие элементы
- **Причина:** вероятно, шрифт пересчитывается кумулятивно (каждый DEAL добавляет размер). Либо ошибка в `_flash_bet_display()` / `sweep_to_max()` — анимация меняет font_size и не возвращает обратно
- **Фикс:** зафиксировать размер шрифта Royal Flush. После любой анимации подсветки — принудительно возвращать font_size к значению из `ui_config.json`. Paytable `custom_minimum_size` должен быть фиксированным
- **Файл:** `game.gd` → `_build_paytable()`, `paytable_display.gd`

### 2.2 Подсказка "Выберите карты и жмите ЗАМЕНА" — позиция
- **Баг:** подсказка пересекается с надписями ВЫИГРЫШ и ПОСЛЕДНИЙ ВЫИГРЫШ
- **Фикс:** опустить подсказку ниже, чтобы она была между полями WIN/LAST WIN и картами, без пересечений. Проверить anchors и y-offset

### 2.3 Мигание кнопки DEAL — поведение
- **Баг:** кнопка DEAL мигает постоянно без остановки
- **Нужно:** 3 пульсации (modulate 0.4↔1.0) через каждые 5 секунд бездействия. После 3 пульсаций — пауза 5 секунд — снова 3 пульсации. НЕ постоянно
- **Файл:** `game.gd` → `_start_idle_blink_timer()` — переписать логику

### 2.4 Кнопка магазина "+" — кликабельность
- **Баг:** значок "+" (переход в магазин) не всегда кликабельный
- **Причина:** вероятно, другой элемент перекрывает зону нажатия (подсказка, бейдж, или z_index)
- **Фикс:** проверить `mouse_filter` всех элементов над кнопкой магазина. Кнопка "+" должна иметь `mouse_filter = MOUSE_FILTER_STOP`, перекрывающие элементы — `MOUSE_FILTER_IGNORE`

### 2.5 TOTAL BET — показать калькуляцию
- **Сейчас:** просто число "TOTAL BET: 400"
- **Нужно:** показать формулу: `TOTAL BET: {chip}100 × 4 = {chip}400`
  - Где `{chip}` — значок фишек (тот же глиф, что используется в BALANCE)
  - `100` = denomination (bet_amount)
  - `4` = bet level (1–5)
  - `400` = итого
- **Формат ключа:** добавить в translations.json: `game.total_bet_calc_fmt` → EN: `"Total Bet: %s × %d = %s"`, RU: `"Общая ставка: %s × %d = %s"`
- **Область:** Single hand пока, но в будущем может распространиться на другие режимы

### 2.6 Окно double — значок фишек у чисел
- **Баг:** в окне подтверждения double числа без значков фишек
- **Фикс:** перед каждым числом (текущий выигрыш и удвоенная сумма) добавить глиф фишки. Формат: `{chip}100 → {chip}200`
- **Файл:** `game.gd` → double overlay, ключ `double.msg_fmt`

### 2.7 Страница правил — выравнивание таблицы
- **Баг:** таблица с описанием машин в окне правил не выровнена, без подложки
- **Фикс:** 
  - Таблицу выровнять по центру
  - Добавить тёмную подложку (аналогично тексту правил выше)
- **Файл:** `game.gd` → `_show_info()`

### 2.8 Баланс — размер шрифта фишек
- **Баг:** поле баланса как-будто поджимается, размер шрифта числа фишек уменьшается. Число фишек не должно быть меньше слова "БАЛАНС"
- **Фикс:** зафиксировать `custom_minimum_size` поля баланса. Число фишек — `size_flags_horizontal = SIZE_EXPAND_FILL`, шрифт — фиксированный из `ui_config.json` → `balance_font_size`. Не позволять контейнеру сжимать текст
- **Файл:** `game.gd` → поле баланса в top bar

# 3. MULTIHAND (Triple / Five / Ten Play)

### 3.1 Надпись "Выигрыш" пропадает после DEAL

- **Фикс:** надпись "Выигрыш: X" (или "WIN: X") должна быть ВИДИМА ВСЕГДА, даже когда выигрыш = 0 или ещё не определён (фаза draw). Показывать "WIN: 0" или "WIN: —"
- **Файл:** `multi_hand_game.gd`

### 3.2 Удалить подсказки "Сделай ставку" и "Выберите карты"
- **Изменение:** из ВСЕХ режимов кроме Single Hand удалить floating-подсказки:
  - "Сделай ставку и жми DEAL"
  - "Выберите карты и жмите ЗАМЕНА"
- **Причина:** эти режимы для профессионалов, подсказки не нужны
- **Область:** `multi_hand_game.gd`, `spin_poker_game.gd` — убрать вызовы `_show_hold_hint()`, `_show_bet_hint()` и аналоги

### 3.3 Локализация — HELD не переводить
- **Баг:** надпись HELD переводится на RU ("ОСТАВ" или другое)
- **Фикс:** HELD должно быть на английском во ВСЕХ языках (аналогично DEAL, DRAW)
- **Файл:** `data/translations.json` → ключ `game.held` → установить `"HELD"` для en, ru, es

### 3.4 Бейджи комбинаций НА руках — фиксированная ширина
- **Баг:** ширина бейджей зависит от длины текста комбинации, прыгает
- **Фикс:** для каждого лейаута рук рассчитать фиксированную ширину:
  - **Экстра руки (маленькие):** ширина бейджа = 80% от ширины одной экстра руки
  - **Основная рука (primary):** ширина бейджа = 33% от ширины основной руки
  - Текст внутри — `text_overrun_behavior = OVERRUN_TRIM_ELLIPSIS` если не влезает
- **Важно:** бейджи ПО БОКАМ от рук (сбоку) — идеальные, их НЕ трогать. Эта задача только про бейджи, которые появляются НА САМИХ руках (поверх карт)
- **Область:** `multi_hand_game.gd` — все лейауты (3/5/10 рук), а также Ultra VP mode

### 3.5 Бейджи комбинаций ПО БОКАМ — увеличить шрифт
- **Изменение:** увеличить font_size бейджей по бокам от рук на 1pt (например с 11 до 12, или с 12 до 13)
- **Файл:** `multi_hand_game.gd` → `_make_badge()`

### 4.4 Множители — привязка к ставке (КРИТИЧНЫЙ БАГ)

  2. Если есть — восстановить его
  3. Если нет — сбросить все множители на 1x
- **Реализация:** в `SaveManager` хранить dict `ultra_multipliers: { "bet5_denom100_hands5": [2,1,3,1,1], ... }`. Ключ = строка из параметров. При смене bet/denom/hands — загружать соответствующий массив
- **Приоритет:** КРИТИЧНЫЙ — abuse экономики

### 4.5 Время инкремента выигрыша — уменьшить на 30%

### 4.6 Контрол бар — кнопка HANDS вместо ULTRA VP

### 4.8 Окно правил — таблица множителей

# 5. SPIN POKER

### 5.1 Ширина автомата — уменьшить

### 5.2 Контрол бар — компактнее к центру
- **Баг:** поля Выигрыш / Общая ставка / Баланс слишком широко разнесены и близко к краям экрана
- **Фикс:** зафиксировать аспект и высоту контролбара. Поля — ближе к центру. Не растягивать на всю ширину экрана. Использовать `custom_minimum_size` + `SIZE_SHRINK_CENTER` для контейнера
- **Файл:** `spin_poker_game.gd`

### 5.3 Окно правил — переработка
- **Изменения:**
  1. **Кнопки линий** — расположить горизонтально СНИЗУ, под автоматом. Убрать слово "LINES" — оставить только номер (1, 2, 3... 20). Ширину кнопок рассчитать так, чтобы все 20 поместились в одну строку
  2. **Иллюстрации линий на автомате** — сделать ярче. Сейчас линии как будто под полупрозрачным окном. Увеличить opacity линий до 1.0, цвет — яркий
  3. **Paytable** — убрать из окна правил (временно, потом придумаем размещение)
  4. **Автоматическое воспроизведение линий** — вернуть: линии должны показываться автоматически одна за другой (auto-cycle). Нажатие на конкретную кнопку линии — стартует auto-cycle с выбранной линии
- **Файл:** `spin_poker_game.gd` → `_show_paytable()`

### 5.4 Убрать лишнюю анимацию переворота перед раундом
- **Баг:** перед началом раунда происходит анимация переворота карт, хотя они уже закрыты
- **Фикс:** убрать flip-анимацию в начале раунда. Карты уже показывают рубашки — не нужно их переворачивать повторно

### 5.5 Закрытие карт — шторки вместо переворота
- **Сейчас:** карты закрываются анимацией переворота (flip)
- **Нужно:** эффект ШТОРОК:
  - Верхняя линия: шторка закрывается СВЕРХУ ВНИЗ
  - Нижняя линия: шторка закрывается СНИЗУ ВВЕРХ
  - Рубашки — это не рубашки карт, а визуальные шторки ПОВЕРХ карт
  - Закрытие последовательно СЛЕВА НАПРАВО (начиная с левого рила): первый рил закрылся на 50% → начинает закрываться второй → и т.д. (overlap/каскад)
  - **Одновременно с закрытием каждого рила** — рил начинает вращение (связано с 5.7)
- **Реализация:** для каждой ячейки — `ClipContent` контейнер с двумя дочерними: карта + шторка. Шторка анимируется через `position.y` (или `size.y`)

### 5.6 Указатели линий — правильная высота
- **Баг:** все 20 указателей линий расположены на уровне центральной линии
- **Нужно:** каждый указатель должен быть на высоте той строки, с которой НАЧИНАЕТСЯ соответствующая линия:
  - Верхняя строка: указатели линий, начинающихся сверху
  - Средняя строка: указатели линий, начинающихся из центра
  - Нижняя строка: указатели линий, начинающихся снизу
  - На каждой стороне по 3 указателя на уровень (кроме нижней левой — там 4)
  - 10 указателей слева, 10 справа
- **Файл:** `spin_poker_game.gd` → `_make_line_ribbon()` — пересмотреть позиционирование

### 5.7 Удалить кнопки BET и BET MAX
- **Изменение:** убрать кнопки BET и BET MAX из Spin Poker. Bet всегда = 1 (фиксированный). Это упрощает UI и соответствует оригиналу
- **Файл:** `spin_poker_game.gd` → контролбар

### 5.8 Анимация барабана рилов (СЛОЖНАЯ ЗАДАЧА)
- **Цель:** превратить Spin Poker в полноценную слот-машину с вращающимися барабанами
- **Описание:** 5 рилов (столбцов) высотой 3 (верхняя, средняя, нижняя строка). Полная аналогия со слот-машиной:

**DEAL-фаза:**
1. Шторки закрываются (5.5) — верхняя и нижняя линия
2. Под шторками начинается вращение: все 3 карты каждого рила прокручиваются вертикально (не рандомная смена, а иллюзия непрерывного движения ленты карт)
3. Рилы останавливаются последовательно слева направо с задержкой ~300ms
4. Остановка: deceleration (ease-out) + bounce (отскок ~5px)
5. После остановки всех рилов — шторки открываются (верхняя вверх, нижняя вниз)

**DRAW-фаза:**
- Незафиксированные столбцы — снова вращение барабана (те же рилы крутятся)
- Зафиксированные — остаются на месте

**Техническая реализация (изучить и выбрать лучший вариант):**

Вариант A — Лента спрайтов:
- Для каждого рила (столбца) создать вертикальную ленту из N спрайтов карт (N = ~15–20 промежуточных + 3 целевых)
- Контейнер с `clip_contents = true`
- Анимация: Tween `position.y` ленты
- При остановке: подставить целевые карты в нужные позиции

Вариант B — Shader:
- Vertical UV scroll shader на каждом столбце
- Текстура-атлас карт, прокрутка UV координат
- Меньше нод, но сложнее контроль отдельных карт

Вариант C — Кольцевой буфер:
- Массив из 5–6 TextureRect на рил, двигаются вниз
- Когда карта уходит за нижний край — телепортируется наверх с новой текстурой
- Создаёт иллюзию бесконечной ленты

**Параметры (в `configs/animations.json`):**
```json
{
  "spin_reel_speed_px_per_sec": 3000,
  "spin_reel_deceleration_ms": 800,
  "spin_reel_bounce_px": 5,
  "spin_reel_column_delay_ms": 300,
  "spin_filler_cards_count": 15,
  "spin_shutter_close_ms": 400,
  "spin_shutter_cascade_offset_ms": 150
}
```

**Приоритет:** HIGH — это ключевая фича, определяющая feel игры
